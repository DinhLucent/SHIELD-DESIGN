## 🏛 Core Mechanisms (ProMax Workflows)

### The Core Asset Protocol
An ironclad set of rules mitigating generic outputs. Whenever an active brand is targeted, the agent conducts exactly 5 mandatory steps:
1. **Query** the user for existing brand guidelines.
2. **Search** official brand press kits and homepages.
3. **Download** foundational assets (Logo, Official Renders, Real UI Screen-grabs).
4. **Extract** exact Hex values natively from DOM/SVGs.
5. **Burn** variables into a rigid `brand-spec.md` preventing hallucinations.

### The Junior Designer Loop
We avoid "zero-to-one" monolithic coding. The system operates via managed phases:
- **Phase 1**: Clarifying questions & factual web search verification.
- **Phase 2**: Broad architectural assumptions and wireframe placeholders shown immediately.
- **Phase 3**: Iteration upon user confirmation (avoiding 100x cost scaling on late reworks).

### Anti-AI Slop Framework
A dedicated engine preventing the "Visual Greatest Common Divisor" seen in standard LLM outputs. 
- **Banned**: Mindless purple gradients, decorative emojis per bullet, generic rounded cards + left block accents, and poorly drawn SVGs.
- **Enforced**: `text-wrap: pretty`, CSS Grid logic, nuanced SERIF pairings, authentic photography, and precise Oklch color spaces.
